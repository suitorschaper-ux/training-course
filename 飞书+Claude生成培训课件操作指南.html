01 飞书端操作

获取飞书会议文字记录

大区总在飞书发起/加入会议，飞书AI会自动实时转写发言内容。会议结束后，进入会议纪要页面，找到「文字记录」标签，定位到业务指导的具体段落。

1
开启飞书会议「妙记」功能 飞书

会议开始前，确认飞书妙记（AI会议助手）已开启录音转写。会议过程中无需手动记录，飞书自动实时生成带时间戳的文字记录。

2
选定范围 · 复制原文 飞书

圈定业务指导的起止时间段（本例：9:55:50 至会议结束），选中该段全部文字记录，复制到剪贴板。无需任何整理，保留原始口语化内容即可。

⌨️操作要点
约 2 分钟
① 点击「文字记录」Tab → ② 找到目标时间戳 → ③ 点击时间戳播放视频确认内容 → ④ 从起始位置选中文字到末尾 → ⑤ Ctrl+C 复制
⚡
不需要提前整理！
飞书的原始文字记录包含口语化表达、重复词、语气词——这些完全没关系，Claude可以识别原始转写内容中的核心逻辑，不需要你先人工清洗。
02 内容理解

理解内容框架，明确课件目标

在把内容交给Claude之前，你需要在脑子里过一遍：这段内容的核心框架是什么？面向谁？要达到什么培训目的？这决定了你给Claude的指令质量。

本次案例：原始会议内容
来源 大区月度会议 · 9:55:50 后的餐饮招商指导段落
时长 约 15 分钟口语内容
内容 外部调研→内部复盘→招商策略→招商计划→保障措施，五大模块
格式 飞书转写的口语化原文，含语气词、重复表达
策划培训课件目标
受众 门店店总、总助、招商人员
目的 让未参会人员理解方法论，指导实际招商工作
格式 HTML课件，可在线浏览、方便分发
风格 专业、视觉化、层次清晰，有图表和结构化内容
🎯
关键认知：你的判断 + Claude的执行 = 最佳结果
Claude负责将内容结构化并美化呈现，但你对业务的理解是不可替代的。 你知道"这是五大模块"、"受众是招商人员"、"要突出操作性"——把这些告诉Claude，它就能生成真正有价值的课件，而不是泛泛的总结文档。
03 Claude 端操作

构建高质量 Prompt

一个好的Prompt = 角色设定 + 内容来源 + 目标受众 + 输出格式 + 特殊要求。这是决定课件质量的最关键一步。

✍️Prompt 模板（可直接复用）
推荐格式
粘贴到 Claude 的完整 Prompt
// ① 说明内容来源 以下是飞书会议的文字记录片段，来源于我们大区月度会议中， 大区总经理在 9小时55分50秒 之后针对餐饮招商的专项指导内容。 // ② 粘贴原始文字记录（直接贴，不用整理） [在此处粘贴从飞书复制的文字记录原文] // ③ 明确任务 请基于以上内容，生成一个面向 店总/总助/招商人员 的培训课件。 // ④ 指定格式 格式要求：HTML 网页格式，要有： - 精美的视觉设计（导航栏、分章节布局、卡片式内容） - 结构化的内容呈现（分模块、有重点标注） - 可以在浏览器中直接打开浏览 // ⑤ 内容要求 内容要求： - 它里面的内容主要涵盖五大板块：外部调研、内部复盘、招商策略、招商计划、保障措施 - 提炼核心方法论，去掉口语化重复表达 - 保留讲话中的关键案例（如：海底捞案例、衢州NEED案例） - 语气专业但易懂，适合一线招商人员理解执行
✅ Prompt 五要素
来源 说明内容来自飞书会议哪个时间段，给Claude提供上下文
受众 明确面向谁——店总/总助/招商人员，决定内容深度和语气
格式 指定HTML/PPT/Word，以及视觉风格要求
结构 告诉Claude内容包含哪几个模块，避免遗漏重要章节
特需 保留哪些案例、用什么语气、有哪些关键词不能改
❌ 常见错误Prompt
❌ 错误示例1
"帮我把这段文字整理成PPT"
→ 缺少受众、格式、结构信息，结果会很平淡
❌ 错误示例2
"先整理一下，再出课件"（两步走）
→ 多此一举，Claude可以直接处理原始转写文字
❌ 错误示例3
"生成一个好看的课件"
→ "好看"没有标准，必须说清楚风格和结构

05 对话式修改

对话式迭代优化

课件生成后，你可以像聊天一样告诉Claude需要修改的地方。不需要重新开始，直接在同一个对话里说"把XXX改成YYY"即可。

🔧 本次迭代过程（真实案例）
📸修改1：纠正品牌名称
30秒
截图发给Claude：「烤鱼 半天要」→ 把"半天要"改为"半天妖"
Claude：已将第02节内部复盘表中"半天要"更正为"半天妖"
✏️修改2：纠正别字
20秒
「偏离了就是补破，不是招商」→ "补破"改为"补铺"
Claude：已将"补破"修正为"补铺"
🏷️修改3：替换品牌名
20秒
全局替换"咪的"→"NEED"（品牌正式名称）
Claude：已替换全文2处"咪的"为"NEED"
🔡修改4：全局调整字号
30秒
"正文字号有点小，把所有正文字体的字号都+3px"
Claude：已将CSS中10px~16px的正文字号全部上调3px
💬 可以这样告诉 Claude
内容修改类
"第X节的XXX内容有误，改为YYY" "把[品牌名]全部替换为[正确名称]" "第03节缺少ZZZ内容，请补充"
样式调整类
"正文字号太小，全部+3px" "取消第X节的空白占位块" "导航栏颜色改深一些"
结构调整类
"在第02节后面增加一个小结模块" "把第04节的表格改成卡片式布局" "最后加一个学习检测问答题"
📸
截图反馈更高效
发现问题时，直接截图标注（如图片中用红框圈出），同时用文字说明要改什么，Claude能更精准定位问题位置。
06 经验总结

最佳实践 & 注意事项

基于本次餐饮招商课件生成的实际经验，总结出以下使用心得，帮助你在下次使用时少走弯路。

🕰️ 传统方式
整理会议笔记 2—3 小时
制作PPT/文档 3—5 小时
设计美化 1—2 小时
合计 6—10 小时
⚡ 飞书 + Claude
飞书复制文字记录 2 分钟
编写Prompt发送 3 分钟
Claude生成 + 迭代修改 5 分钟
合计 10 分钟
✅ 让结果更好的做法
  • 会议前确认飞书妙记已开启，全程录音转写
  • 指导完成后立刻处理，趁记忆鲜活校对内容
  • Prompt中明确说出内容包含的模块数量
  • 把关键案例名称（NEED、海底捞）在Prompt里提一次
  • 遇到错字/专有名词，截图+文字说明同时发给Claude
  • 生成后快速通读，把模糊表达反馈给Claude完善
❌ 避免的做法
  • 等到几天后才整理，遗忘关键语境
  • 先人工整理再给Claude，浪费时间
  • Prompt只说"帮我做个课件"不给细节
  • 遇到错误就重新开对话，应在同一对话迭代
  • 忽略口语化内容中的关键案例细节
  • 生成后直接分发，不做基本内容校对
